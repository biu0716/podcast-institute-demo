# 零摩擦获取全文：实现说明（给 codex）

> 目标：让用户尽量不手动粘贴文稿。两条路并行：
> **方案1 剪贴板自动捕获**（用户只在小宇宙 App 里复制，助手自动收）
> **方案2 视频平台免费字幕**（节目同时在 YouTube/B站时，直接抓字幕，连复制都免了）
>
> 红线：不逆向小宇宙私有接口、不自动操控其 App 抓取。只被动接收用户主动复制的内容。

---

## 方案 1：剪贴板自动捕获

### 用户体感
1. 用户在某单集页面点「📋 从小宇宙导入文稿」。
2. 助手弹提示：「请打开小宇宙 App → 该单集 → 文稿 → 长按全选 → 复制。复制好后回到这里，我会自动接收。」并**开始监听剪贴板**。
3. 用户在 App 里复制文稿，切回助手。
4. 助手检测到剪贴板里出现一大段中文长文本 → 自动填入、停止监听、开始处理。
5. 用户**全程不需要回来手动粘贴或点确认**（可保留一个"识别到文稿，确认使用？"的轻确认，防误捕）。

### 技术实现（server.py，本机运行）

监听用 macOS 自带 `pbpaste`，无需第三方库：

```python
import subprocess, time, threading

def read_clipboard() -> str:
    try:
        return subprocess.run(["pbpaste"], capture_output=True, text=True, timeout=2).stdout
    except Exception:
        return ""

def looks_like_transcript(text: str) -> bool:
    if not text:
        return False
    t = text.strip()
    # 文稿特征：足够长 + 以中文为主 + 多句
    if len(t) < 400:                      # 阈值按实测调，文稿通常上千字
        return False
    cjk = sum('一' <= c <= '鿿' for c in t)
    if cjk / max(len(t), 1) < 0.3:        # 中文占比，过滤英文/代码/URL
        return False
    if t.count('。') + t.count('，') < 5:  # 多标点 = 连续语句
        return False
    return True

def watch_clipboard(session_id, on_capture, timeout_sec=120, poll=0.8):
    """用户点'去复制'后调用。捕到文稿或超时即停。"""
    baseline = read_clipboard()           # 记录起始内容，只认'新出现'的
    start = time.time()
    while time.time() - start < timeout_sec:
        cur = read_clipboard()
        if cur and cur != baseline and looks_like_transcript(cur):
            on_capture(session_id, cur)   # 回填 + 入 transcript 缓存
            return
        time.sleep(poll)
    on_capture(session_id, None)          # 超时，前端提示可手动粘贴兜底
```

前端通过一个轻量轮询接口（如 `/clipboard/status?session=...`）拿状态：`listening / captured / timeout`，捕到就把文本展示出来让用户确认。

### 边界与隐私（必须做）
- **只在用户主动触发后监听**，绝不常驻后台读剪贴板。
- **拿到目标即停**；设 120 秒超时自动停。
- **只记 baseline 之后的新内容**，避免误用用户之前复制的无关文字。
- 文稿**只存到本地 transcript 缓存**，不上传任何第三方。
- 跨平台：Windows 用 `powershell Get-Clipboard`，Linux 用 `xclip -o`；非 macOS 时降级为手动粘贴框。
- 兜底：超时或判定失败时，前端始终保留传统粘贴框，不让用户卡死。

### 入缓存
捕获的文稿按 `episode_id` 写入 transcript 缓存，`source = user_paste`，与 ASR/官方文稿同等对待，后续专题直接复用，永不重复要求。

## 方案 2：视频平台免费字幕（yt-dlp）

### 思路
不少播客同时发在 YouTube / B站，那里常有**免费自动字幕**。能抓到就完全跳过小宇宙文稿，连复制都省了，0 成本拿全文。

### 流程（pipeline.py）
```
拿到单集 → 提取节目名+单集标题
  → 在 YouTube / B站 搜同名内容
  → 命中且时长接近(±时长容差) → yt-dlp 抓字幕
  → 清洗成纯文本 → 入 transcript 缓存(source=video_caption)
  → 没命中 → 回落到剪贴板/Groq 转写
```

### 抓字幕命令

YouTube（优先人工字幕，无则自动字幕；中文优先）：
```bash
yt-dlp --skip-download \
  --write-subs --write-auto-subs \
  --sub-langs "zh-Hans,zh,zh-CN,en" \
  --sub-format "vtt/srt/best" \
  -o "%(id)s.%(ext)s" \
  "<video_url>"
```

B站（哔哩哔哩，yt-dlp 支持，部分视频有 AI 字幕）：
```bash
yt-dlp --skip-download --write-subs --write-auto-subs \
  --sub-langs "ai-zh,zh-Hans,zh" --sub-format "srt/best" \
  -o "%(id)s.%(ext)s" "<bilibili_url>"
```

先用 `--list-subs` 探测有哪些语言/类型再决定抓哪个：
```bash
yt-dlp --list-subs --skip-download "<url>"
```

### 字幕清洗（VTT/SRT → 纯文本）
```python
import re

def vtt_to_text(raw: str) -> str:
    lines = []
    for ln in raw.splitlines():
        ln = ln.strip()
        if (not ln or ln == "WEBVTT" or "-->" in ln
                or ln.isdigit() or ln.startswith(("Kind:", "Language:"))):
            continue
        ln = re.sub(r"<[^>]+>", "", ln)        # 去内联时间戳标签
        lines.append(ln)
    # 自动字幕常有重复行（滚动效果），去连续重复
    out = []
    for ln in lines:
        if not out or out[-1] != ln:
            out.append(ln)
    return "\n".join(out)
```

### 匹配可靠性（重要，别配错节目）
- 标题模糊匹配 + **时长比对**（视频时长 vs 音频时长，差异超 ~10% 判为不同内容）。
- 命中不确定时，标 `confidence=medium` 并在专题里注明"文稿来自视频平台字幕，可能与音频版本有出入"。
- 自动字幕质量参差，`source=video_caption` 的可信度低于 `official`/`asr`，在来源徽标上区分对待。

### 入缓存
同样按 `episode_id` 写 transcript 缓存，`source=video_caption`，复用规则一致。

## 两条路在主流水线里的位置

更新获取全文的优先级（成本/摩擦从低到高）：

```
1. transcript 缓存命中            → 0 成本 0 操作   ✅
2. 视频平台字幕(yt-dlp)           → 0 成本 0 操作   ← 方案2，新增，优先于让用户动手
3. 剪贴板捕获官方文稿              → 0 成本 1 个复制动作  ← 方案1
4. Groq 全文转写                  → ~$0.04/集，几秒
5. 本地 faster-whisper 后台       → 0 成本，慢，兜底额度
```

注意顺序：**方案 2 排在让用户复制之前**——能自动抓字幕就别麻烦用户。方案 1 是"有官方文稿且无视频版"时的低摩擦选项。
