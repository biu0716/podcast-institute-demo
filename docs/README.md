# 🎧 播客专题研究助手 · Podcast Topic Researcher

> 贴一个**问题**或**播客链接**，自动把多期播客整理成一篇可读、可检索、可分享的专题文章。
> 与单集摘要工具不同——它的核心是**跨多期交叉比对**：找出不同主播之间的共识、分歧与独到观点，并为每个结论标注证据来源。

## ✨ 为什么不一样

市面上的播客 AI 工具基本都停在**单集摘要**。这个项目想解决的是更难、也更有价值的事：

> 当你想搞懂一个话题，往往要听好几期不同节目。这工具帮你把它们**交叉读完**——
> 谁和谁意见一致、在哪里吵起来、分歧的根源是什么、谁提出了别人没有的视角。

输出的核心是一张**观点矩阵**：

| 子议题 \ 来源 | 主播 A | 嘉宾 B | 主播 C | 关系 |
|---|---|---|---|---|
| AI 会取代初级程序员吗 | 会，2 年内 🎧 | 不会，只是工具变化 🎧 | 看领域 📝 | ⚠️ 分歧 |
| 现在该不该学 AI 编程 | 必须学 🎧 | 必须学 🎧 | 必须学 🎧 | ✅ 共识 |
| 提示工程会消失吗 | — | 会消失 🎧 | — | 💡 独有 |

> 徽标 🎧 听稿 · 🎬 视频字幕 · 📝 简介 · 🌐 联网 · 🤔 推断 —— 每个结论都能追溯它的证据强度。

## 🧠 核心特性

- **跨集交叉比对** — 不是把几篇摘要拼一起，而是「抽取观点 → 聚类议题 → 对齐立场 → 判定共识/分歧」的四段流水线。
- **来源可信度体系** — 每个论断标注来源徽标，区分「主播确实说过」与「从简介推断」，绝不把 AI 推断写成事实。
- **先快后全** — 1–3 分钟先给基于公开资料的初稿，再后台补全文证据，自动升级。
- **多来源 + 自动降级** — 缓存 → 视频字幕 → 云端转写 → 本地转写，永远有免费兜底。
- **本地优先、隐私友好** — 数据都在本机，使用你自己的 API key，不上传第三方。
- **可读可检索可分享** — 网页书架、书内全文搜索、选中文本联网检索、划线生成金句图。

## 🏗️ 架构

```
输入（问题 / 链接）
      │
      ├─ 检索 & 选片（播放量 / 影响力 / 新鲜度）
      ▼
  获取全文（命中即停，成本/摩擦从低到高）
      ├─ ① transcript 缓存          0 成本 0 操作
      ├─ ② 视频字幕 (yt-dlp)         0 成本 0 操作
      ├─ ③ Groq 全文转写            ~$0.04/集（默认）
      └─ ④ 本地 faster-whisper      0 成本（兜底）
      ▼
  跨集比对流水线  A 抽取 → B 聚类 → C 对齐 → D 成稿
      ▼
  专题网页（观点矩阵 + 文章 + 来源徽标）→ 本地书架
```

详见 [DESIGN.md](./DESIGN.md)。

## 🚀 快速开始

```bash
git clone https://github.com/<you>/podcast-topic-researcher.git
cd podcast-topic-researcher
pip install -r requirements.txt        # groq, faster-whisper, yt-dlp ...

cp .env.example .env                   # 填入你的 GROQ_API_KEY（可选，没有则走本地转写）
python server.py                       # 启动本地服务
# 打开 http://127.0.0.1:8765/
```

依赖：Python 3.10+、ffmpeg（读音频时长 / 切片）、yt-dlp（抓视频字幕）。

## ⚙️ 配置

| 变量 | 说明 | 默认 |
|---|---|---|
| `GROQ_API_KEY` | Groq 转写用，留空则只用本地 Whisper | — |
| `TRANSCRIBE_BACKEND` | `groq` / `local` / `auto` | `auto` |
| `WHISPER_MODEL` | 本地模型大小 `small`/`medium` | `small` |
| `PORT` | 本地服务端口 | `8765` |

## 🔒 数据来源与合规

本项目**只使用合法来源**：

- 用户自己提供的播客**公开音频**做转写（用户自己的 API key / 本地算力）；
- **公开视频平台**的字幕（yt-dlp）；
- 用户**主动粘贴**的文本；
- 节目的公开页面信息（简介、Show Notes、时间轴等）。

**不逆向任何平台私有接口，不自动化操控 App 抓取受保护内容。** AI 生成的专题会明确标注每段内容的来源与证据强度，不冒充原文。
